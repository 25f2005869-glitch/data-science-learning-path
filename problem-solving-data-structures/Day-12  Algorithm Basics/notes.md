# Day 12 – Algorithm Basics

## Objective

The goal of this session is to understand what an algorithm is and why algorithms are important in problem solving.

---

## What is an Algorithm?

An algorithm is a step-by-step procedure used to solve a problem or perform a task.

An algorithm should be:

* Correct
* Clear
* Finite
* Efficient

---

## Example: Making Tea

1. Boil water.
2. Add tea leaves.
3. Add sugar.
4. Add milk.
5. Serve tea.

This is an algorithm because it consists of ordered steps.

---

## Characteristics of an Algorithm

### Input

Data provided to the algorithm.

Example:

```text id="alg01"
Two numbers
```

### Output

Result produced by the algorithm.

Example:

```text id="alg02"
Sum of two numbers
```

### Finiteness

The algorithm must end after a finite number of steps.

### Effectiveness

Each step should be simple and executable.

---

## Example Algorithm: Sum of Two Numbers

Input:

```text id="alg03"
10, 20
```

Algorithm:

1. Read first number.
2. Read second number.
3. Add both numbers.
4. Display the result.

Output:

```text id="alg04"
30
```

---

## Pseudocode

Pseudocode is an informal way of writing algorithms.

Example:

```text id="alg05"
START

READ A
READ B

SUM = A + B

PRINT SUM

END
```
